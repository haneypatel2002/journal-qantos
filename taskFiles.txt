============================================================
Journal Qntos
AI-Powered Journaling & 21-Day Challenge App
React Native (Expo)
============================================================

PROJECT OVERVIEW

Journal Qntos is an AI-powered journaling and self-improvement mobile application built using React Native with Expo.

The purpose of this application is to help users:

- Write daily journal entries
- Track moods day-wise
- Visualize emotional patterns using a GitHub-style graph
- Receive personalized 21-day improvement challenges
- Track daily task progress visually
- Improve emotional awareness, focus, and personal growth

All data is currently stored locally using AsyncStorage.
Future phases will introduce subscription-based premium features.


============================================================
APPLICATION ROADMAP (3 PHASES)
============================================================


==============================
PHASE 1 – JOURNAL & MOOD TRACKING
(Local Mode – No Login Required)
==============================

OBJECTIVE:
Allow users to use the application without login and maintain daily journal entries with mood tracking.

USER FLOW:

1. User opens the app.
2. User enters their name (no authentication required).
3. User is redirected to the main dashboard.
4. User can:
   - Select a date using a calendar slider.
   - Write a journal entry for that day.
   - Select their mood for that day.
   - Save the entry locally.
   - Edit previous entries.

FEATURES TO IMPLEMENT:

- Onboarding screen (Enter Name Only)
- Local user profile stored in AsyncStorage
- Calendar slider for day-wise journal navigation
- Journal text input area
- Mood selection options:
    Happy
    Calm
    Neutral
    Sad
    Angry
    Anxious
- Save and update journal entries
- Daily streak counter
- Entry statistics
- Monthly overview

GITHUB-STYLE MOOD GRAPH:

- Display daily mood entries in a GitHub-style grid
- Color intensity based on mood
- Monthly contribution view
- Dynamic updates when new entries are saved

DATA STRUCTURE (LOCAL STORAGE):

Journal Entry Object:
- id
- date
- mood
- content
- timestamp

All entries stored in AsyncStorage as JSON.


==============================
PHASE 2 – AI-BASED 21-DAY CHALLENGE SYSTEM
==============================

OBJECTIVE:
Analyze journal mood patterns and suggest improvement plans.

MOOD ANALYSIS LOGIC:

Based on stored journal moods, generate suggestion cards such as:

- Feel Better Card
- Focus Card
- Self Improvement Card
- Meditation Card
- Productivity Boost Card

Initial implementation can use rule-based logic.
Future enhancement can integrate AI API for deep analysis.

CHALLENGE FLOW:

1. User sees suggested improvement cards.
2. User selects a challenge.
3. A 21-day challenge begins.
4. Each day unlocks a scratch card.
5. Scratch card reveals the daily task.
6. User marks task as completed.
7. Progress updates in a 21-day GitHub-style graph.

SCRATCH CARD SYSTEM:

- Day-wise locked/unlocked system
- Scratch animation effect
- Daily activity description
- Completion toggle
- Store progress locally

CHALLENGE DATA STRUCTURE:

Challenge Object:
- id
- category
- startDate
- progress (array of 21 days)
- completedDays
- status

Store challenge data in AsyncStorage.

21-DAY PROGRESS GRAPH:

- Separate Git-style graph for challenge progress
- Shows completed vs incomplete days
- Auto-updates daily
